File extensions

On this page:

  • Supported mesh files
  • Supported field files
  • All files
  • Customized extensions

The application Intermediate - file extensions shows how to change the file extensions expected when the Open file widget is showed. It also explains the use of type and subtype attributes, continuing with the content learned in Beginner: leaves tutorial.

The XML file of this example can be found at <installdir>/apps/i-Intermediate - field visualization/a-New/local.mnu.xml, where <installdir> is the installation folder.

Supported mesh files

To allow to choose a mesh file supported by OpenNum you have to use the attribute subtype="mesh". 

<leaf name="Mesh file" type="file" subtype="mesh">
</leaf>

All files

To select any file type you have to use the attribute subtype="data". 

<leaf name="Mesh file" type="file" subtype="data">
</leaf>


Figure 3: All file extensions.

Customized extensions

With the attribute subtype="data:[...]" you can also define a wildcard to customize the allowed file extension showed in the Open file widget.

The expected string format after data: can be explained as follows in pseudocode: 

Description 1|Extension 1;Extension 2|Description 2|Extension 3 ... 

<leaf name="Mesh file" type="file" subtype="data:All supported (*.txt;*.csv)|*.txt;*.csf| Plain text file (*.txt)|*.txt| Comma separated values (*.csv)|*.csv">
</leaf>


Figure 4: Customized file extensions.

Structs

The tag struct is used as a container for other elements, it can contain different type of childs like leafs or other structs. Graphically, it allows display a set of widgets keeping the data hierarchy.

This example shows how to build a widget hierarchy using the tag struct. This example describes the attribute selection which displays two different types of widgets as occurs with the tag leaf.

Single selection

The tag struct with the attribute selection="single" displays a combo box that allows to keep selected one between several excluding options. The sample code teaches how to build a widget of this type. 

<struct name="value" selection="single">
	<leaf name="Float 1" type="float">
	</leaf>
	<leaf name="Float 2" type="float">
	</leaf>
</struct>


Figure 1:Single selection widget.

No selection

The tag struct with the attribute selection="none" displays a box with several options, this options represents different ways in the data hierarchy. selection="single" is the default behavior of the struct tag. The sample code teaches how to build a widget of this type. 

<struct name="value" selection="none">
	<leaf name="Float 1" type="float">
	</leaf>
	<leaf name="Float 2" type="float">
	</leaf>
</struct>


Figure 2:No selection widget.

Multiple level structs

The main objective of the struct tag is to allow building hierarchys of leaf and struct elements. Every struct element can contain a set of leaf and struct childs. Leaf elements means the end of a way for a branch of the hierarchy, but struct child elements can contain more leafs or structs. The following example describes a tow branch hierarchy. 

<struct name="Struct 1: level 1">
	<struct name="Struct 1: level 2">
		<leaf name="Float 1" type="float">
		</leaf>
		<leaf name="Float 2" type="float">
		</leaf>
	</struct>
	<struct name="Struct 2: level 2">
		<struct name="Struct 1: level 3">
			< name="Text 1" type="char">
			</leaf>
		</struct>
	</struct>
</struct>


Figure 3:Multiple level struct.

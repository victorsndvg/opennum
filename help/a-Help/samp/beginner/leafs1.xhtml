Leafs

The tag leaf is the last element in the XML hierarchy for showing grafical components. This tag is used to display widgets which allow you to enter different types of data, the type of data and widget is selected by the required attibute type.

This example shows how to create some widgets to introduce different data types. Show some posible values for the atribute type, mandatory for leaf tag, which defines the look, behavior and data type of the widget.

Float box

The tag leaf with the attribute type="float" allows to enter a single float or a list floats separated by blanks or commas, you can use parentheses or brackets to delimit the list. The following code creates a float widget. 

<leaf name="float-variable" type="float">
</leaf>

To write a evenly spaced list of floats you can use the Matlab® format (start:step:end).


Figure 1:List of floats in Matlab® format.

Complex box

The tag leaf with the attribute type="complex" allows to enter a single complex or a list complex. The complex supported format is (float,float) where the blanks are not important, otherwise the string is not processed. The following code creates a complex widget. 

<leaf name="complex-variable" type="complex">
</leaf>


Figure 2: Complex widget with a list of complex.

Text box

The tag leaf with the attribute type="char" allows to enter a single line string. The following code creates a text widget. 

<leaf name="text-variable" type="char">
</leaf>

File box

The tag leaf with the attribute type="file" creates a file widget. This widget contains a button to display an open file dialog. 

<leaf name="file-path" type="file">
</leaf>

Folder box

The tag leaf with the attribute type="folder" creates a folder widget. This widget contains a button to display an open folder dialog. 

<leaf name="folder-path" type="folder">
</leaf>

Fundamental elements of OpenNum

On this page:

  • Working folder
  • Application
  • Sample data

The first step to work with OpenNum is to choose a working folder. Then, one of the available applications must me associated with the working folder. Eventually, a sample data can be load. These concepts are explained below.

Working folder

Before running any scientific solver, a working folder must be selected. A working folder will manage a single execution of a scientific solver. Thus, it is the natural place to save all the data and results files of the execution. Nevertheless, files associated with the working folder could be saved outside and they would be accessed using their path relative to the working folder.

To select a working folder, go to menu Project - Select folder... (see Figure 1) and select a working folder. The last working folder used in the previous session will be automatically opened when you start OpenNum.


Figure 1: Menu item Select folder....

Application

OpenNum can provide an interface to several scientific solvers. When two solvers require the same data and produce the same type of outputs, they can share the same menu items. That would be the case, for example, of two solvers that solve the same problem but using two different internal algorithms. Thus, we would say that both solvers belong to the same application; more specifically, an application is a solver or set of solvers that share the same menu configuration.

After selecting a working folder, the user must associate an application to it. To do so, go to menu Project - Application (see Figure 2) and select one of the existing applications. If there is only one application, menu Application will be disabled and the application will be automatically associated to the working folder when the latter is selected.


Figure 2: Menu item Application.

After selecting an application, a warning appears to inform you that the files of the previous application associated to the working folder, if exists, will be erased (see Figure 3).


Figure 3: Warning dialog box.

Sample data

An application can have several sample data associated to it. Each sample data contains a set of data files and outputs that exemplify the use of the application. That would be the case, for example, of an application named "Direct current" with two sample data: the first one containing the simulation of an electric motor, and the second one containing the simulation of an electric distribution network.

To load a sample data, go to menu Project - Sample data (see Figure 4) and select one of the existing sample data. If there is only one sample data available for the current application, menu Sample data will be disabled and the sample data will be automatically load when the associated application is selected.


Figure 4: Menu item Sample data.

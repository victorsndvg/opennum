The Visualization window is the part of the interface where the graphics are shown. It is composed of several parts, as we see in Figure 1. These parts are described in detail below.


Figure 1: Parts of the visualization window.

Graphic tags

This part shows a tag for every graphic that is being visualized (see Figure 2). The user can change the current graphic by simply clicking in a tag.


Figure 2: Graphic tags.

Main bar

This bar controls some aspects of the visualization (see Figure 3). It also controls whether the Scale bar and the Parameter bar are visible or not.


Figure 3: Main bar.

The main bar contains the following buttons:

  • Save...: it allows us to choose a filename to save a snapshot of the Graphic scene. In the case of a Plot over line graphic, two filenames are required to save each part of the Graphic scene. The valid formats are:
    • JPEG, PNG, BMP, TIFF as raster formats;
    • PDF, EPS, PS, SVN as vector formats. The latter will only be available if option VTK_USE_GL2PS was active when configuring the compilation of VTK library.
  • Perspective: establishes a specific perspective. There are four options: XY view, XZ view, YZ view and iso view. This button is unavailable in graphs.
  • Center: centers the graphic in the Graphic scene. This button is unavailable in graphs.
  • Names: when active, this button shows the field name as it appears in the file. This button is unavailable in graphs.
  • Legend: when active, this button shows the Legend.
  • Outline: when active, this button shows the Outline. This button is unavailable in graphs.
  • Wireframe/Surface: it allows us to choose the visualization mode. For scalar fields there are three options: Wireframe (show field only on edges), Surface (show field on surfaces) and Surface+Edges (show edges and field on surfaces). For vector fields there are three options: Wireframe (show edges), Surface (show surfaces) and None (do not show any part of the domain).
  • Mesh: when active, it shows edges on the mesh. This button is only available in the graphic References.
  • Plane: when active, it shows the cut plane. This button is only available in the graphics Slice, Cut and Rough cut.
  • Labels: controls the visibility of labels. In the graphics the graphics Numbering and References there are three options: No labels, Selected (show labels only for the selected entities) and All (show labels for all entities). This button is only available in the graphics Numbering and References. In the graphic 2D graph, when is active, shows the graph legend.
  • Edges: when active, shows edges in the field. This button is only available in the graphic Contour.
  • Table: when active, shows the materials that compound the domain. This button is only available in the graphic Materials.
  • Colored/Plain: it allows us to choose the visualization mode in the Scalar deformed graphic. There are two options: Colored (shows the field on the deformed domain), Plain (does not show the field).
  • Components: controls the component to show in the Graphic Vector Components. There are four options: X comp., Y comp., Z comp. and Modulus.
  • Scale bar: when active, shows the Scale bar.
  • Parameter bar: when active, shows the Parameter bar, where parameter is the name of the parameter (time, frequency,...).
  • Opacity bar: when active, allow us to define the percentage of actors opacity.

Graphic scene

Here is where the visualizations are shown. It also contains the Outline, the Legend and the Scale (see Figure 4).


Figure 4: Graphic scene.

Mouse interaction

This graphics scene can be altered through mouse events:

  • Rotate: drag with the left mouse button.
  • Zoom: drag with the right mouse button.
  • Move: drag with the central mouse button.
  • Double-click: in some scalar and vector field-related graphics, a double-click shows the value of the field.

In graphics Slice, Cut and Rough cut we recommend to hide the plane through the Plane button in the Main bar before using the previous mouse events.

Scale bar

This bar controls some aspects related to the color scale (see Figure 8). It contains the following buttons:


Figure 8: Scale bar.

  • Scale: when active, shows the Scale in the Graphic scene.
  • Absolute/Relative scale: indicates how to calculate the extreme values of the field. There are two options: Absolute scale (extreme values are calculated in the whole domain) and Relative scale (they are calculated in the portion of the domain that is visible in the Graphic scene). This button is only available in the graphics Slice, Cut and Rough cut.
  • Adjust: it allows us to adjust the limits of the scale range; possible options are:
    • Adjust manually: the user can change the limits manually.
    • Adjust to every field: the limits will be the extreme values of the field; in the case of a parameter-dependent simulation, this limits will be recalculated for every parameter step.
    • Adjust to the current field: the limits will be the extreme values of the current field. In the case of a parameter-dependent simulation, this limits will not be recalculated for every parameter step.
    • Adjust to the parameter interval: In the case of a parameter-dependent simulation, the limits will be the extreme values of all parameter steps. Here parameter refers to the name of the parameter (time, frequency,...).
  • From/To: when the user wants to adjust manually the scale range, these buttons allow him to introduce the range limits.

Parameter bar

This bar controls some aspects of the visualization of parameter-dependent fields (see Figure 9). Here parameter is the name of a particular parameter (time, frequency,...).


Figure 9: Parameter bar.

The bar includes the following buttons:

  • First frame "|<": shows the first frame of the series.
  • Previous frame "<|": shows the previous frame of the series.
  • Play ">": plays the series.
  • Pause "||": pauses the animation.
  • Next frame "|>": shows the next frame of the series.
  • First frame ">|": shows the last frame of the series.
  • Parameter: shows/edits the current parameter value in the animation.
  • Start: shows/edits the start parameter value in the animation.
  • End: shows/edits the end parameter value in the animation.
  • Duration: shows/edits the duration in seconds of the animation.

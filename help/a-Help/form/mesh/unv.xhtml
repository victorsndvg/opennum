UNV format

The Universal File formats were originally developed by the Structural Dynamics Research Corporation (SDRC) in the late 1960s and early 1970s to facilitate computer aided engineering (CAE). This format is described in the web page of the Structural Dynamics Research Lab at the University of Cincinnati (UC-SDRL).

In the UNV format, meshes are composed by datasets. The original Universal File formats are composed of several thousands of different datasets, but the relevant datasets for OpenNum are:

  • dataset 2411: describe the node coordinates;
  • dataset 2412: describe global indexing of the nodes;
  • dataset 2467: describe the groups of nodes, edges and faces.

The UNV format only admits Lagrange P1 and P2 finite elements.

OpenNum supports a subset of FE descriptor IDs of dataset 2411. These finite elements types must be transformed to a VTK cell type to be managed by the graphical interface; we write in parentheses the VTK cell type that become after being read:

  • 11: Rod (3: VTK_LINE)
  • 41: Plane Stress Linear Triangle (5: VTK_TRIANGLE)
  • 44: Plane Stress Linear Quadrilateral (9: VTK_QUAD)
  • 111: Solid Linear Tetrahedron (10: VTK_TETRA)
  • 112: Solid Linear Wedge (13: VTK_WEDGE)
  • 115: Solid Linear Brick (12: VTK_HEXAHEDRON)

OpenNum identifies the entities of a group defined in the dataset 2467 by the group number instead of the group name. Thus, format UNV is interchangeable with format MFM in the meshes where both formats are available. In order to manage the reference numbering of nodes, edges, faces and elements, the graphical interface create some integer fields explained in the VTK+ format.

OpenNum expects extension .unv for files written in Universal File format. OpenNum has been tested with .unv files constructed by Salome.

Attribute tooltip

The elements struct and leaf can have the optional attribute tooltip="value", that indicates a string to appear as a tooltip when the user hovers the pointer over a widget, without clicking it.

<struct name="cond" title="Existing conditions:" tooltip="You can add, delete or edit the conditions">
  ...
</struct>

Attributes helpdata and helpconfig

The elements struct and leaf can have the optional attribute helpdata="value", that shows a new button Help in the widget. value can be an HTML code, an URL, or a local file. The optional attribute helpconfig="mode, width, height, title" configures the help window:

  • mode sets how to interpret value; there are three possibilities:
    • url indicates that value is an URL when it starts by http:// and a local file otherwise;
    • urlinst indicates that value is a file with a path relative to the installation folder;
    • code indicates that value is HTML code;
  • width and height set the width and height in pixels of the help window (set -1 to get the default values);
  • title sets the window title.

If the attribute helpconfig is missing, OpenNum takes helpconfig="url, -1, -1, Help".

<leaf name="one" type="float" helpdata="http://www.usc.es/">
  ...
</leaf>

<struct name="cuatro" helpdata="file.png" helpconfig="url,200,200,Image">
  ...
</struct>

<struct name="tres" helpdata="help/interface/doc.html" helpconfig="urlinst,300,200,Document">
  ...
</struct>

<leaf name="two" type="float" helpdata="<b>HTML Help!</b>" helpconfig="code,400,300,Help">
  ...
</leaf>

Introducing files

An element leaf with attribute type="file" manages files through this widget. This element can be empty. To save an initial value, write the filename in an element element. Two sub-types can be defined:

  • Mesh files: When the element also has the attribute subtype="mesh", manages finite element mesh files. The previous attribute can indicate the permitted file types and extensions using subtype="mesh: desc | extension list | desc | extension list | ...", where
    • desc is a description of a file type and
    • extension list is a list of extensions associated to this file type, separated by semicolons; each extension must be preceded by *.

    Files must have one of the valid mesh formats in case you want to visualize them with OpenNum .

    <leaf type="file" name="Mesh file" subtype="mesh:All supported (*.mfm;*.unv)|*.mfm;*.unv| 
  Modulef Formatted Meshes (*.mfm)|*.mfm| Universal files (*.unv)|*.unv">
</leaf>
  • Field files: When the element also has the attribute subtype="field", manages finite element field files. The previous attribute can indicate the permitted file types and extensions using subtype="mesh: desc | extension list | desc | extension list | ...", where
    • desc is a description of a file type and
    • extension list is a list of extensions associated to this file type, separated by semicolons; each extension must be preceded by *.

    Files must have one of the valid field formats in case you want to visualize them with OpenNum .

    <leaf type="file" name="Field file" subtype="field:All supported (*.vtk;*.vtu)|*.vtk;*.vtu|
Visualization Toolkit (VTK) (*.vtk)|*.vtk|Visualization Toolkit (VTU) (*.vtu)|*.vtu">
</leaf>

For both subtypes mesh and field, an optional attribute mesh_name="prefix:value" permits to show value as the text label in a visualization, when the button Labels is active. Here prefix can be:

  • data, and then value is a string scalar;
  • menu, and then value is a menu path, as it is explained in section Dependencies.

When attribute mesh_name does not appear, the text label is the filename.

Introducing folders

An element leaf with attribute type="folder" manages folders through this widget. This element can be empty. To save an initial value, write the folder path in an element element.

<leaf" type="folder" name="Folder for outputs>
  <element>../output</element>
</leaf>

Selecting string scalars

An element leaf with attribute type="char" manages a string scalar through this widget. This element can be empty. To save an initial value, write every single data in an element element.

<leaf" type="char" name="Heat expression>
    <element>x^2+^2-z</element>
</leaf>

<leaf name="temp" type="char" title="Temperature expression (°C)">
</leaf>

Selecting string arrays

An element leaf with attribute type="charlist" manages string arrays. The user cannot change the element content, just select it. If the content must change, you can use instead an element struct with attribute customizable. The selection can be single or multiple depending on the attribute selection:

  • Single selection: requires the attribute selection="single" (this is the default value); the string array is shown through this widget. If one element element has the attribute selected="true", the string that encloses will appear as pre-selected. The last string selected by the user will take the atribute selected="true". 
    <leaf type="charlist" name="Boundary type">
  <element>Isolated</element>
  <element selected="true">Conductor</element>
</leaf>
  • Multiple selection: requires the attribute selection="multiple"; the string array is shown through this widget. If one or several elements element have the attribute selected="true", the strings that enclose will appear as pre-selected. The last strings selected by the user will take the atribute selected="true". 
    <leaf type="charlist" name="Isolated surfaces" selection="multiple">
  <element>Up</element>
  <element selected="true">Down</element>
  <element selected="true">Right</element>
  <element>Left</element>
</leaf>

Visualization of plain text files

An element leaf with attributes type="charlist" and showfile="file:filename" shows the content of the file filename through this widget.

<leaf type="charlist" showfile="file:output.txt" title="Results:">
</leaf>

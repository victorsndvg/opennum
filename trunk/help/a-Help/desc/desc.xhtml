Introduction

OpenNum is an application designed to distribute scientific solvers with a nice graphical interface.

Typically, to program a GUI is time consuming and requires experience with graphic libraries. Once the interface is done, a small modification can involve changes at the low-level programming. OpenNum lets you create a graphical interface adapted to your solvers by simply editing an XML configuration file.

More specifically, OpenNum lets you

  • to collect a hierarchical dataset,
  • to call any executable file and
  • to visualize scalar and vector fields, plot graphs or show simple plain text files.

It also has other useful utilities specifically designed for numerical simulation packages:

  • it allows managing a centralized materials database;
  • it can read several finite element mesh formats and several field formats.

OpenNum is well adapted to work with numerical simulation solvers; hence its name is an acronym for Open Numerical (simulation) where open refers to its GPL license.

Roles

OpenNum can be used by two different types of people ("roles"):

  • distributors are those who want to distribute their own executable files; they have to adapt OpenNum to their specific requirements, creating suitable menus in the interface. A distributor must understand the internal organization of OpenNum, so a carefully reading of this help is recommended.
  • users are those who want to use a set of scientific solvers packed within OpenNum by a distributor. An user only needs to know the usage of the graphical interface.

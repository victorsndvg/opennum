Description

A package can be distributed with a materials database, common for all the applications. A materials database can store the physical properties of a set of materials. That is useful when working with scientific solvers that simulate a physical phenomenon: the thermal behaviour of a motor, the elastic deformation of a bridge, etc. In those cases, several physical properties must be defined and associated with each part of the domain. The materials database permits to do such associaciation in two steps:

  1. the physical properties are defined for several materials in the materials database.
  2. a material is associated with each part of the domain using a specific process designed by the distributor (for example, allowing the user to choose the materials in a menu item).

Folder location

The materials database distributed with the package must be located on <installdir>/materialsDB, where <installdir> is the installation folder. As it was comented in the section Renaming the materials database the name of the materials database can be materials.xml or materials.<name>.xml where <name> is every package name.

With the first execution of the graphical interface, OpenNum creates a configuration folder in the user's HOME and saves there a modifiable copy of the materials database in the subfolder materialsDB. The user could modify this copy through the graphical interface.

Every time an executable file is called from the graphical interface, a copy of the latter materials database is saved in the working folder with the name materials.dat.xml. Thus, the executable file can rely on that a copy of the lastest modified materials database exists in the working folder.

Materials database file

The complete menu associated to a materials database consists of a tree composed of menu items, submenu items, structures and leafs. Leafs can contain several data types. This information is coded using XML format, a metalanguage developted by W3C.

In the XML format, each block of information, or element, is delimited by a start-tag and a end-tag. For example, the start- and end-tags for element struct are <struct> and </struct>. The characters between the start- and end-tags, if any, are the element's content, and may contain markup, including other elements, which are called child elements.

Elements in XML can have attributes, that is, a name/value pair. For example, the element struct can have the attribute selection with the value single: <struct selection="single">.

The complete list of elements and attributes allowed in the menu's description is in the Section XML syntaxis.

The materials database file is coded in utf-8.

Executable files

The distributor must store all the executable files called from the graphical interface in the folder <installdir>/solvers, where <installdir> is the installation folder. The executable files can be called from the graphical interface through an action mark.

In some cases, the user would want to manage his own version of the executable files, even if he does not have write permission in the installation folder. This is the case when:

  • the distributor provide source code for the executable files and the user wants/has to compile himself the executable files;
  • the user wants to remotely run the executable files so he has to save them in the external server.

To cover those cases, OpenNum follows this policy:

  1. When a executable file is called from the graphical interface, OpenNum looks for the file in the solvers subfolder of the configuration folder.
  2. If the file does not exists in the previous location, OpenNum looks for the file in <installdir>/solvers, where <installdir> is the installation folder.

Thus, an user can store the executable files in a subfolder of his HOME and still work with OpenNum.

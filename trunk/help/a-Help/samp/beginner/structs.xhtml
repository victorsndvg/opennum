Structs

On this page:

  • Single selection
  • No selection
  • Multiple level structs

The application Beginner - structs is an example that shows how to manage struct elements; these elements can contain other leaf and struct elements, allowing us to keep a tree structure in the data hierarchy. This example also describes the selection attribute to display two different widgets, as occurs in the leaf element.

The XML file of this example can be found at <installdir>/apps/a-Beginner - structs/a-New/local.mnu.xml, where <installdir> is the installation folder.

Single selection

A struct element displays a combo box; every line in the combo box shows the name attribute of a child element. The user can select one of the child elements and this selection is recorded in the data file. The following example produces a widget of this type, as it is shown in Figure 1:

<struct name="value" selection="single">
    <leaf name="Float 1" type="float">
    </leaf>
    <leaf name="Float 2" type="float">
    </leaf>
</struct>

Since single selection is the default value, to add the selection="single" attribute does not change the behavior of the widget but it can be useful to remark it.

No selection

A struct with selection="none" attribute displays a box; every line in the box shows the name attribute of a child element. The user can click on every line in order to browse the data tree hierarchy. The following example produce a widget of this type, as it is shown in Figure 2:

<struct name="value" selection="none">
    <leaf name="Float 1" type="float">
    </leaf>
    <leaf name="Float 2" type="float">
    </leaf>
</struct>


Figure 2: No selection widget.

Multiple level structs

As we explained above, the main goal of the struct element is to organize data as a tree. Thus, struct element are nodes of the tree whereas leaf elements are terminal symbols. The following example shows a data tree with several hierarchical levels.

<struct name="Struct 1: level 1">
    <struct name="Struct 1: level 2">
        <leaf name="Float 1" type="float">
        </leaf>
        <leaf name="Float 2" type="float">
        </leaf>
    </struct>
    <struct name="Struct 2: level 2">
        <struct name="Struct 1: level 3">
            < name="Text 1" type="char">
            </leaf>
        </struct>
    </struct>
</struct>

As we can see in Figure 3, every time the user clicks on a child element, its associated widget appears below the parent widget.


Figure 3: Multiple level struct.

Leaves with lists

On this page:

  • Single selection
  • Multiple selection
  • Show file

The application Beginner - char list leaves is an example that shows how to manage widgets with string lists. These widgets can be used, not only for data input, but also to choose single or multiple selections from a list of options. This example also shows the use of selection, showfile and type="charlist" attributes.

The XML file of this example can be found at <installdir>/apps/a-Beginner - char list leaves/a-New/local.mnu.xml, where <installdir> is the installation folder.

Single selection

A leaf element with type="charlist" attribute displays a combo box to select one option from a list. This example shows how to build a widget of this type, as it is shown in Figure 1.

<leaf name="Charlist" type="charlist" selection="single">
    <element>Element 1</element>
    <element>Element 2</element>
    <element>Element 3</element>
</leaf>

Since single selection is the default value, to add the selection="single" attribute does not change the behavior of the widget but it can be useful to remark it.

Multiple selection

A leaf element with type="charlist" and and selection="multiple" attributes displays a box to select several options from a list. This example shows how to build a widget of this type, as it is shown in Figure 2.

<leaf name="Charlist" type="charlist" selection="multiple">
    <element>Element 1</element>
    <element>Element 2</element>
    <element>Element 3</element>
</leaf>


Figure 2: Multiple selection widget.

Show file

A leaf element with type="charlist" and showfile="prefix:filename" shows the content of the plain-text file filename, as explained in the XML syntax of leaves. The following example shows this behavior, as it is shown in Figure 3.

<leaf name="plain text file" type="charlist" showfile="file:file.txt">
</leaf>


Figure 3: Plain text widget.

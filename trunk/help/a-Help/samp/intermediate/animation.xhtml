Animation

On this page:

  • Time dependant animation
  • Variable dependant animation
  • Interpolation

The application Intermediate - animation shows how to visualize the parameter depending evolution of a field. This means that you can view animations with OpenNum. It also explains the use of evolution attribute to assign the name of the evolving parameter and the interpolation attribute to do linear interpolation between two adjacent existing results.

To display an animation you must use PVD (Paraview Data) files in the mesh attribute, this is the default file format for ParaView visualizer. The field file set pointed by the PVD file must contain any VTK file type supported by OpenNum (polygonal, uniform rectilinear, nonuniform rectilinear, curvilinear, or unstructured).

The XML file of this example can be found at <installdir>/apps/m-Intermediate - animation/a-New/local.mnu.xml, where <installdir> is the installation folder.

Time dependant animation

The default behavior of OpenNum while working with PVD files is to show a time animation bar. This behavior can be forced using the attribute evolution="time". 

<struct name="Time animation" plot="vector_field" mesh="file:animation/animation.pvd" pointdata="data:Magnetic field">
</struct>

Variable dependant animation

To visualize variable dependant animations you have to use the attribute evolution="variable_name". 

<struct name="Frequency animation"  evolution="frequency" plot="vector_field" mesh="file:animation/animation.pvd" pointdata="data:Frequency field">
</struct>


Figure 2: Animation bar.

Interpolation

With the attribute interpolation="true", the default behavior, you can visualize results by interpolating two adjacent fields in the same PVD file. Use the animation bar to introduce the value in the first text box.

To block this behaviour you can use the attribute interpolation="false". 

<struct name="Frequency animation"  evolution="frequency" interpolation="false" plot="vector_field" mesh="file:animation/animation.pvd" pointdata="data:Frequency field">
</struct>

The application Advanced - variables is an example that shows how to use dependencies in order to reference existing values along the menu. It may be usefull to recall some sintax concepts collected in section dependencies. Variables can be used with the attributes that contain menu: and file: prefixes.

The XML file of this example can be found at <installdir>/apps/s-Advanced - variables/a-New/local.mnu.xml, where <installdir> is the installation folder.

Single level variables

You can use variables to reference one or more values specified in the XML hierarchy. If a variable returns several strings the result will be evaluated one time for every returned value.

The following sample code shows how to list some content placed in the menu structure. It allows to access to stored values at the same level hierarchy after a brach.

<leaf name="String list" type="charlist" source="menu:/List/${/List}">
</leaf>

In this case we assume the existence of two leaves placed in the paths /List/Leaf 1 and /List/Leaf 2, after applying the variable will get a list of both contents.

Multiple level variables

You can use multiple levels of variables, these will be solved recursively from the innermost level to the outermost.

The following sample code shows how to retrieve information from two branches on three hierarchy levels straight.

<leaf name="List all descriptions" title="Descriptions list:" type="charlist" source="menu:/Data/${/Data/${/Data}}">
</leaf>

Variables and files

Variables can be used to reference files along with the prefix file:. This can be very useful to reference output files generated from data entered in the menu.

The following sample uses a variable with a relative path to build the correct name of a mesh file.

<struct name="Plot" type="charlist" mesh="file:${../Select media}.vtu" plot="mesh">
</struct>

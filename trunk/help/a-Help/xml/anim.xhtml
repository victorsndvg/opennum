Animation

On this page:

  • The attribute evolution
  • The attribute interpolation

Given any of the graphics explained in the sections Mesh visualization and Field visualization, the visualization can be animated by indicating a PVD file in the attribute mesh. Besides, we can decide

  • the type of interpolation by using the attribute interpolation;
  • the name of the animation parameter by using the attribute evolution.

The attribute evolution

The attribute evolution="value" indicates that the field depend on the parameter value (for example, time, frequency, etc.). The name of the parameter serves to rename the Parameter bar and some buttons within.

The attribute interpolation

The attribute interpolation="value" determines how the interpolation must be performed in the animation. value can be

  • true: (this is the default value) a linear interpolation will be performed in the parameter interval and the step length will be proportional to the length of the parameter interval; besides, values indicated in the Parameter button of the Parameter bar could be anyone; 
    <struct name="Example" mesh="file:heat.pvd" celldata="data:temperature" plot="filled" 
evolution="Time" interpolation="true"></struct>
  • false: there is no interpolation in the parameter interval and the step length will be constant; besides, values indicated in the Parameter button of the Parameter bar could only be those saved in the PVD file.
    • <struct name="Example" mesh="file:fluid.pvd" celldata="data:pressure" plot="threshold" 
evolution="Frequency" interpolation="false"></struct>
